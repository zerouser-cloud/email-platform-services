<task type="auto">
  <name>Task 2: Verify gateway unified error format (VER-04)</name>
  <files></files>
  <read_first>packages/foundation/src/errors/grpc-to-http.filter.ts</read_first>
  <action>
Test that the gateway returns errors in the unified format `{ statusCode, message, error, correlationId, timestamp }`.

The GrpcToHttpExceptionFilter is registered as a global filter on the gateway. It handles both HttpException and gRPC errors. Since there are no proxy routes yet (GrpcClientsModule is a TODO stub), test the error format by:

1. Hit a non-existent route (triggers NestJS NotFoundException which is an HttpException):
   ```
   curl -s http://localhost:4000/api/nonexistent | python3 -m json.tool
   ```
   Expected response shape:
   ```json
   {
     "statusCode": 404,
     "message": "Cannot GET /api/nonexistent",
     "error": "Not Found",
     "correlationId": "<uuid-or-no-correlation-id>",
     "timestamp": "<ISO-8601>"
   }
   ```

2. Verify all required fields are present:
   ```
   curl -s http://localhost:4000/api/nonexistent | python3 -c "
   import sys, json
   d = json.load(sys.stdin)
   required = ['statusCode', 'message', 'error', 'correlationId', 'timestamp']
   missing = [f for f in required if f not in d]
   if missing:
       print(f'FAIL: missing fields: {missing}')
       sys.exit(1)
   print(f'statusCode={d[\"statusCode\"]}')
   print(f'message={d[\"message\"]}')
   print(f'error={d[\"error\"]}')
   print(f'correlationId={d[\"correlationId\"]}')
   print(f'timestamp={d[\"timestamp\"]}')
   print('ALL_FIELDS_PRESENT')
   "
   ```

3. Verify HTTP status code is 404:
   ```
   curl -s -o /dev/null -w "%{http_code}" http://localhost:4000/api/nonexistent
   ```
   Expected: `404`

4. Test a POST to verify method handling:
   ```
   curl -s -X POST http://localhost:4000/api/nonexistent | python3 -c "
   import sys, json
   d = json.load(sys.stdin)
   assert 'statusCode' in d and 'correlationId' in d and 'timestamp' in d
   print('POST_ERROR_FORMAT_OK')
   "
   ```

IMPORTANT: This is verification only. Do NOT modify any source files.
  </action>
  <verify>
    <automated>curl -s http://localhost:4000/api/nonexistent | python3 -c "import sys,json; d=json.load(sys.stdin); assert all(k in d for k in ['statusCode','message','error','correlationId','timestamp']); print('UNIFIED_ERROR_FORMAT_OK')"</automated>
  </verify>
  <acceptance_criteria>
    - Response contains all 5 required fields: statusCode, message, error, correlationId, timestamp
    - statusCode is a number (404 for not found)
    - correlationId is a string (UUID or 'no-correlation-id')
    - timestamp is an ISO-8601 string
    - error is a human-readable error category string
  </acceptance_criteria>
