<task type="auto">
  <name>Task 1: Start infrastructure and services via docker compose (VER-01, VER-02)</name>
  <files></files>
  <read_first>infra/docker-compose.yml, .env.docker, package.json</read_first>
  <action>
Run the full docker compose stack including builds:

1. First, clean any previous state:
   ```
   docker compose -f infra/docker-compose.yml down -v 2>/dev/null || true
   ```

2. Start the full stack with build (this is what `pnpm docker:up` does but detached):
   ```
   docker compose -f infra/docker-compose.yml up -d --build
   ```

3. Wait for containers to become healthy (up to 120 seconds):
   ```
   docker compose -f infra/docker-compose.yml ps --format json
   ```
   Poll every 10 seconds until all 10 containers show status "healthy" or "running".

4. Verify infrastructure containers (VER-01):
   Check that these 4 are healthy:
   - mongodb (mongo:7)
   - redis (redis:7-alpine)
   - rabbitmq (rabbitmq:3-management)
   - minio (minio/minio)

5. Verify service containers (VER-02):
   Check that these 6 are healthy:
   - gateway (port 4000 mapped to 3000)
   - auth (ports 3001, 50051)
   - sender (ports 3002, 50052)
   - parser (ports 3003, 50053)
   - audience (ports 3004, 50054)
   - notifier (port 3005)

6. Check service logs for runtime errors:
   ```
   docker compose -f infra/docker-compose.yml logs --tail=50 gateway auth sender parser audience notifier 2>&1 | grep -iE "error|exception|fatal|crash" || echo "NO_ERRORS_FOUND"
   ```
   Lines containing "error" in structured log field names (like "error_message" key) are acceptable.
   Lines containing actual stack traces, unhandled exceptions, or "Bootstrap failed" are NOT acceptable.

IMPORTANT: This is verification only. Do NOT modify any source files. If a container fails to start, report the exact error from `docker compose logs <service>` and stop.
  </action>
  <verify>
    <automated>docker compose -f infra/docker-compose.yml ps --format "table {{.Name}}\t{{.Status}}" | grep -c "(healthy)" | grep -q "10" && echo "ALL_HEALTHY" || echo "NOT_ALL_HEALTHY"</automated>
  </verify>
  <acceptance_criteria>
    - All 4 infrastructure containers (mongodb, redis, rabbitmq, minio) report "healthy"
    - All 6 service containers (gateway, auth, sender, parser, audience, notifier) report "healthy"
    - No "Bootstrap failed", unhandled exception, or fatal error in service logs
  </acceptance_criteria>
