  <name>Task 1: Deploy application to DEV environment via Coolify</name>
  <read_first>
    - .planning/phases/18-deployment-via-coolify/18-RESEARCH.md (Steps 4-5)
    - .planning/phases/18-deployment-via-coolify/18-02-SUMMARY.md (infrastructure hostnames)
  </read_first>
  <action>
    Create the Docker Compose application resource in Coolify for the DEV environment.

    **Step A: Create Docker Compose Resource (per D-03)**
    1. In Coolify, go to Project "Email Platform" -> `dev` environment
    2. **+ Add Resource** -> **Private Repository (via GitHub App)**
    3. Select the GitHub App you connected in Plan 02
    4. Select repository: `email-platform-services`
    5. Branch: `dev` (per D-04)
    6. Build pack: **Docker Compose**
    7. Docker Compose file path: `docker-compose.prod.yml`

    **Step B: Enable Predefined Network**
    1. In the resource settings, find **"Connect to Predefined Network"**
    2. Enable it -- this allows the compose stack to reach Coolify-managed PostgreSQL, Redis, RabbitMQ, Garage
    3. WITHOUT this, services will get "connection refused" to infrastructure

    **Step C: Configure Domain (per D-09, D-10, D-12)**
    1. In the resource settings, find the **Domains** section for the `gateway` service
    2. Set domain: `https://api.dev.email-platform.pp.ua`
    3. The `https://` prefix tells Traefik to auto-provision TLS via Let's Encrypt (per D-12)
    4. Port: `3000` (gateway listens on 3000 inside the container)

    **Step D: Configure Environment Variables (per D-05)**
    Set these env vars in the Docker Compose resource settings. Replace `<hostname>` placeholders with actual Coolify hostnames from Plan 02.

    ```
    # Image tag for dev branch
    IMAGE_TAG=dev-latest

    # HTTP Ports
    GATEWAY_PORT=3000
    AUTH_PORT=3001
    SENDER_PORT=3002
    PARSER_PORT=3003
    AUDIENCE_PORT=3004
    NOTIFIER_PORT=3005

    # gRPC URLs (compose service names -- same stack)
    AUTH_GRPC_URL=auth:50051
    SENDER_GRPC_URL=sender:50052
    PARSER_GRPC_URL=parser:50053
    AUDIENCE_GRPC_URL=audience:50054

    # Infrastructure (use Coolify hostnames from Plan 02)
    DATABASE_URL=postgresql://<user>:<pass>@<dev-postgres-hostname>:5432/email_platform
    POSTGRES_USER=<user>
    POSTGRES_PASSWORD=<pass>
    POSTGRES_DB=email_platform
    REDIS_URL=redis://<dev-redis-hostname>:6379
    RABBITMQ_URL=amqp://<dev-rabbitmq-hostname>:5672
    STORAGE_ENDPOINT=<dev-garage-hostname>
    STORAGE_PORT=3900
    STORAGE_ACCESS_KEY=<garage-access-key>
    STORAGE_SECRET_KEY=<garage-secret-key>
    STORAGE_BUCKET=email-platform
    STORAGE_REGION=garage

    # Cross-cutting
    LOG_LEVEL=debug
    LOG_FORMAT=json
    CORS_ORIGINS=https://dev.email-platform.pp.ua
    CORS_STRICT=true
    PROTO_DIR=/app/proto
    GRPC_DEADLINE_MS=5000
    RATE_LIMIT_BURST_TTL=1000
    RATE_LIMIT_BURST_LIMIT=10
    RATE_LIMIT_SUSTAINED_TTL=60000
    RATE_LIMIT_SUSTAINED_LIMIT=100
    ```

    Note on Garage: If Garage access key/secret are not yet created, deploy first, then exec into Garage container to create:
    ```
    docker exec -it <garage-container> /garage key create email-platform-key
    docker exec -it <garage-container> /garage bucket create email-platform
    docker exec -it <garage-container> /garage bucket allow email-platform --read --write --key email-platform-key
    ```

    **Step E: Deploy**
    1. Click **Deploy** in Coolify
    2. Watch the deployment log for all 6 services starting
    3. Wait for all containers to show healthy status

    **Step F: Note the Coolify Webhook URL**
    After deployment, go to the resource **Settings** and find/copy the **Webhook URL**. You'll need it for GitHub Actions secrets. Also note the **API Token** from Coolify settings (Security -> API Tokens -> generate one if needed).
  </action>
  <verify>

    2. `curl -s https://api.dev.email-platform.pp.ua/health/live` returns 200
