2. **Simplify liveness** (per D-07):

   ```typescript
   @HealthCheck()

     return this.health.check([]);
   }
   ```

3. **Parallelize readiness** (per D-01, D-02):
   Replace the current sequential readiness method with parallel execution using `Promise.allSettled` (not `Promise.all` -- allSettled gives full visibility when multiple services are down, per Research open question 1):

   ```typescript
   @Get(HEALTH.READY)
   @HealthCheck()
   async readiness() {
     const results = await Promise.allSettled(
       this.grpcServices.map(({ key, url }) =>
         this.grpc.checkService<GrpcOptions>(key, key, {
           url,
           timeout: HEALTH.CHECK_TIMEOUT,
           healthServiceCheck: checkOverallHealth,
         }),
       ),
     );

     return this.health.check(
       results.map((result) => () => {
         if (result.status === 'fulfilled') {
           return result.value;
         }
         throw result.reason;
       }),
     );
   }
   ```

   This approach:
   - Runs all 4 gRPC checks concurrently (worst case = 1 timeout period = 3s per D-02)
   - Collects ALL results (both success and failure) via allSettled
   - Wraps each result in a thunk for terminus `health.check()` to aggregate
   - Failed checks throw their original error so terminus reports them correctly
   - All services appear in the response even when some are down

4. Keep the `checkOverallHealth` helper function and `grpcServices` array unchanged -- only the readiness method changes.

5. Keep `@SkipThrottle()` decorator on the class.

**health.module.ts**: No changes needed (it does not reference MemoryHealthIndicator).
