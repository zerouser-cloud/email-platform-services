<objective>
Simplify liveness probes and fix readiness to check only actual per-service dependencies.

Purpose: All 4 backend services (auth, sender, parser, audience) currently inject all 3 indicators (Mongo, Redis, RabbitMQ) regardless of actual dependencies. Notifier uses a local old-pattern indicator instead of the foundation one. Liveness checks heap memory which causes unnecessary pod restarts. Fix each service to check only its real dependencies in readiness, and simplify liveness to always return 200.

Output: Each service has correct liveness (empty) and readiness (actual deps only). Notifier consolidated to use foundation indicator.
</objective>
