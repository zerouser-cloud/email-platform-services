2. **Add `getRetryConfig` helper function** (per D-05) above `retryConnect`. This function merges: explicit options > env vars > defaults. Use `process.env` directly (NOT ConfigService -- this utility runs before DI bootstrap). Implementation:
   ```typescript
   function getRetryConfig(options?: Partial<RetryOptions>): RetryOptions {
     return {
       maxRetries: options?.maxRetries
         ?? parseInt(process.env.RETRY_MAX_RETRIES ?? '', 10)
         || RETRY_DEFAULTS.maxRetries,
       baseDelayMs: options?.baseDelayMs
         ?? parseInt(process.env.RETRY_BASE_DELAY_MS ?? '', 10)
         || RETRY_DEFAULTS.baseDelayMs,
       maxDelayMs: options?.maxDelayMs
         ?? parseInt(process.env.RETRY_MAX_DELAY_MS ?? '', 10)
         || RETRY_DEFAULTS.maxDelayMs,
     };
   }
   ```
   Note: Using `||` (not `??`) after parseInt is intentional -- 0 and NaN are both invalid for retry values, so fallback to defaults is correct.

3. **Update `retryConnect` function** to use `getRetryConfig`:
   - Replace `const { maxRetries, baseDelayMs, maxDelayMs } = { ...RETRY_DEFAULTS, ...options };`
   - With `const { maxRetries, baseDelayMs, maxDelayMs } = getRetryConfig(options);`

4. **Add jitter to delay calculation** (per D-04):

   - With: `const delay = Math.min(baseDelayMs * 2 ** (attempt - 1), maxDelayMs) + Math.floor(Math.random() * baseDelayMs);`
