    2. Remove `NODE_ENV` from the `GlobalEnv` type.

       ```typescript
       CORS_STRICT: z.coerce.boolean().default(false),
       ```
       Update the refine to:
       ```typescript
       .refine(

           message: 'CORS_ORIGINS cannot be "*" when CORS_STRICT is enabled. Specify explicit origins.',
           path: ['CORS_ORIGINS'],
         },
       )
       ```
       Add `CORS_STRICT` to the `GlobalEnv` type as `CORS_STRICT: boolean`.

    4. Keep `PROTO_DIR` in the schema as optional (it is set by Dockerfile, read by proto-resolver).

    **Step B: Sync all three env files** (per D-09, D-10):

    All three files must have the SAME set of keys. Differences are ONLY in values. The canonical key set is:

    ```
    # HTTP Ports
    GATEWAY_PORT, AUTH_PORT, SENDER_PORT, PARSER_PORT, AUDIENCE_PORT, NOTIFIER_PORT

    # gRPC URLs
    AUTH_GRPC_URL, SENDER_GRPC_URL, PARSER_GRPC_URL, AUDIENCE_GRPC_URL

    # Infrastructure
    DATABASE_URL, POSTGRES_USER, POSTGRES_PASSWORD, POSTGRES_DB
    REDIS_URL, RABBITMQ_URL
    MINIO_ENDPOINT, MINIO_PORT, MINIO_ACCESS_KEY, MINIO_SECRET_KEY

    # Cross-Cutting
    LOG_LEVEL, LOG_FORMAT, CORS_ORIGINS, CORS_STRICT

    # Production Hardening
    GRPC_DEADLINE_MS
    RATE_LIMIT_BURST_TTL, RATE_LIMIT_BURST_LIMIT
    RATE_LIMIT_SUSTAINED_TTL, RATE_LIMIT_SUSTAINED_LIMIT
    ```

    Note: `PROTO_DIR` is NOT in `.env` (per D-08) because local dev uses the contracts package path directly. It IS in `.env.docker` and `.env.example` because Docker needs it. This is the one allowed difference -- PROTO_DIR is Docker-only. Document this with a comment in `.env.example`.

    - gRPC URLs: `0.0.0.0:<port>` (local) vs `<service>:<port>` (docker)
    - DATABASE_URL hostname: `localhost` vs `postgres`
    - REDIS_URL hostname: `localhost` vs `redis`
    - RABBITMQ_URL hostname: `localhost` vs `rabbitmq`
    - MINIO_ENDPOINT: `localhost` vs `minio`
    - LOG_FORMAT: `pretty` (local) vs `json` (docker)
    - CORS_STRICT: `false` (both dev modes)
    - PROTO_DIR: absent in .env, `/app/proto` in .env.docker

    - Remove `NODE_ENV` from `.env.docker` and `.env.example` (not present in .env)
    - Remove `PROTO_DIR` from `.env` only (keep in .env.docker and .env.example)
