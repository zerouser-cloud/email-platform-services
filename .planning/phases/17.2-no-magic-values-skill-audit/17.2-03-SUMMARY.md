## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Fixed @Inject() string literals that would break with Symbol tokens**
- **Found during:** Task 1 (Convert DI string tokens to Symbols)
- **Issue:** 10 @Inject() decorators across use-cases and grpc-servers used hardcoded string values ('UserRepositoryPort', etc.) instead of the exported named constants. Converting module tokens to Symbol() without updating these would cause DI resolution failures at runtime.
- **Fix:** Added imports of the named constant from each service's module file and replaced string literals in @Inject() decorators
- **Files modified:** 10 files (use-cases and grpc-servers across all 5 services)
- **Verification:** grep confirms no @Inject('...') string patterns remain in src/
- **Committed in:** e833eaf (part of Task 1 commit)

**Impact on plan:** Essential for correctness -- Symbol tokens require reference equality, string @Inject would silently fail at runtime. No scope creep.
