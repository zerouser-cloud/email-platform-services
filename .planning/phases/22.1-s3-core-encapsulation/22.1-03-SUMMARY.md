## Decisions Made

- **Retain `main` + `types` alongside `exports`:** Some legacy bundlers still look for `main`/`types`. Under `moduleResolution: node16`, `exports` shadows them automatically. No conflict, strictly additive safety.
- **`node16`, not `nodenext` or `bundler`:** `nodenext` is a moving target and could silently flip semantics on a TypeScript upgrade. `bundler` has a documented CJS-mismatch warning in TS PR #62320. `node16` is stable, well-documented, and honors the exports field — exactly what this plan needs.
- **Single-commit atomic bundle (Task 2):** Adding the exports field alone does nothing (legacy resolution ignores it); upgrading tsconfig alone does nothing (no exports field for TS to honor). Both must land in the same commit for HEAD to be meaningfully different at the resolution level.
- **No `./internal/*` wildcard:** Only the single barrel `./internal` resolves. Deep paths are intentionally unreachable to keep the surface tight. Can be widened later if a future plan needs a deep-path import; narrowing after the fact would be harder because the wildcard would already be published.
- **Force rebuild after tsconfig change:** Turbo's default input set may or may not include `tsconfig.base.json` at the repo root. Rather than spelunking Turbo's input hashing internals, the plan mandates `pnpm turbo run build --force` — a 5-second safety margin that guarantees no stale dist/ survives.
