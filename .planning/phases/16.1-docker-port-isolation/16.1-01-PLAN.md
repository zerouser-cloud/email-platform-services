1. Edit `infra/docker-compose.infra.yml`: Remove the `ports:` directive from ALL four services (postgres, redis, rabbitmq, minio). Keep everything else identical -- healthchecks, volumes, networks, environment, restart policy. The services remain accessible to other Docker containers on the `infra` network via their container ports, but are NOT exposed to the host.

2. Create `infra/docker-compose.dev-ports.yml` as a Docker Compose override file. This file re-adds host port mappings for local development when used with `-f` flag. Content:

```yaml
# Dev override: exposes infra ports to host for local development.
# Usage: docker compose -f docker-compose.infra.yml -f docker-compose.dev-ports.yml up
services:
  postgres:
    ports: ["5432:5432"]
  redis:
    ports: ["6379:6379"]
  rabbitmq:
    ports: ["5672:5672", "15672:15672"]
  minio:
    ports: ["9000:9000", "9001:9001"]
```

No networks, volumes, or other keys needed in the override -- Docker Compose merges only the specified fields into the base file.

IMPORTANT per infrastructure-guard skill: Standard ports preserved exactly (5432, 6379, 5672, 15672, 9000, 9001). No new services added. No credentials changed. No healthchecks modified.
