**1. Create `packages/config/src/compose.ts`** (per D-07):

```typescript
import { z, type ZodRawShape } from 'zod';

/**
 * Composes multiple Zod object schemas into a single flat ZodObject via shape spread.
 * Refines on input schemas are NOT preserved -- apply refines after composition.
 */
export function composeSchemas<T extends z.ZodObject<ZodRawShape>[]>(
  ...schemas: T
): z.ZodObject<ZodRawShape> {
  const merged: ZodRawShape = {};
  for (const schema of schemas) {
    Object.assign(merged, schema.shape);
  }
  return z.object(merged);
}
```

Note: The generic return type is simplified to `z.ZodObject<ZodRawShape>`. This is intentional -- exact field-level inference across variadic generics is complex and unnecessary since consumers use `z.infer<typeof MySchema>` on their per-service composed schema, not the utility return type.

**3. Refactor `packages/config/src/config-loader.ts`** (per D-09):

Replace entire file. `loadConfig(schema)` is the only API — no wrapper functions.

```typescript
import { z } from 'zod';

const cache = new Map<z.ZodType, unknown>();

/**
 * Validates process.env against the provided Zod schema.
 * Results are cached by schema reference for idempotency.
 * Works because each service passes the same module-level const schema reference.
 *
 * @param schema - A ZodObject or ZodEffects (refined ZodObject) to validate against
 * @returns Parsed and validated config object
 */
export function loadConfig<T extends z.ZodType>(schema: T): z.infer<T> {
  const cached = cache.get(schema);
  if (cached) return cached as z.infer<T>;

  const result = schema.parse(process.env);
  cache.set(schema, result);
  return result as z.infer<T>;
}
```

No `loadGlobalConfig()` — each service calls `loadConfig()` with its own schema directly. No wrapper, no unnecessary abstraction.
