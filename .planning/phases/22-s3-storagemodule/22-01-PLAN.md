**storage.service.ts** — per D-01, D-02, D-04, D-14:
S3StorageService class that implements StoragePort. Constructor receives S3Client and bucket string (not via DI -- instantiated in provider factory). Methods:
- `upload`: PutObjectCommand with Bucket, Key, Body, ContentType
- `download`: GetObjectCommand, convert Body via `transformToByteArray()`, return `Buffer.from(bytes)`
- `delete`: DeleteObjectCommand with Bucket, Key
- `exists`: HeadObjectCommand, catch error where `name === 'NotFound'` returns false, rethrow others
- `getSignedUrl`: Use `getSignedUrl` from `@aws-sdk/s3-request-presigner` with GetObjectCommand, expiresIn = `Math.ceil(expiresInMs / 1000)` (SDK takes seconds)

**storage.providers.ts** — per D-14, D-15, D-19:
`storageProviders(options: StorageModuleOptions): Provider[]` factory function.
- S3Client provider (provide: S3_CLIENT): Inject ConfigService, construct endpoint URL as `` `http://${config.get<string>('STORAGE_ENDPOINT')}:${config.get<number>('STORAGE_PORT')}` ``, set region from `STORAGE_REGION`, credentials from `STORAGE_ACCESS_KEY`/`STORAGE_SECRET_KEY`, set `forcePathStyle: S3_DEFAULTS.FORCE_PATH_STYLE`, `requestChecksumCalculation: S3_DEFAULTS.REQUEST_CHECKSUM`, `responseChecksumValidation: S3_DEFAULTS.RESPONSE_CHECKSUM`, `maxAttempts: S3_DEFAULTS.MAX_ATTEMPTS`.
- StorageService provider (provide: options.token): Inject S3_CLIENT, create `new S3StorageService(client, options.bucket)`.
- Health provider (provide: STORAGE_HEALTH, useExisting: S3HealthIndicator).
- Return array: [s3ClientProvider, storageServiceProvider, S3HealthIndicator, healthProvider, S3ShutdownService].

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| App -> S3 API | Credentials cross boundary; S3 endpoint may be internal (MinIO) or external (Garage) |
| Presigned URL -> Client | Temporary access to private objects without credentials |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-22-01 | Information Disclosure | presigned URLs | mitigate | Mandatory expiresInMs parameter (D-02), no permanent URLs allowed by interface |
| T-22-02 | Elevation of Privilege | raw S3 client | mitigate | S3_CLIENT and S3StorageService not exported from barrel (D-04), only StoragePort abstraction |
| T-22-03 | Information Disclosure | credentials | mitigate | Credentials only from env vars via ConfigService, never logged or passed to clients |
| T-22-04 | Tampering | bucket access | accept | Per-service bucket isolation (D-07) limits blast radius; IAM policies are infra-level, not code |
</threat_model>
