# Phase 22: S3 StorageModule - Research

**Researched:** 2026-04-09
**Domain:** S3-compatible object storage integration (AWS SDK v3 + NestJS DI)
**Confidence:** HIGH

## Summary

StorageModule follows the established CacheModule pattern: `forRootAsync()` with options, Symbol DI tokens, health indicator, shutdown service. The S3 client uses AWS SDK v3 (`@aws-sdk/client-s3` + `@aws-sdk/s3-request-presigner`), configured with `forcePathStyle: true` and `requestChecksumCalculation: 'WHEN_REQUIRED'` for MinIO/Garage compatibility.

The env vars (`STORAGE_ENDPOINT`, `STORAGE_PORT`, `STORAGE_ACCESS_KEY`, `STORAGE_SECRET_KEY`, `STORAGE_BUCKET`, `STORAGE_REGION`) already exist in `.env` and `.env.docker`, and `StorageSchema` is already defined in `packages/config/src/schemas/storage.ts`. Parser and notifier env schemas already compose `StorageSchema`. S3-03 (env var rename) is already done per CONTEXT.md D-12.

**Primary recommendation:** Mirror CacheModule structure exactly. StorageModule in `packages/foundation/src/storage/`, ReportsStorageModule either co-located or in a subdirectory. Per-service storage modules in `apps/*/src/infrastructure/storage/`. Add `@aws-sdk/client-s3` and `@aws-sdk/s3-request-presigner` to foundation's dependencies.

### Pattern 1: StorageModule.forRootAsync (mirrors CacheModule)

**What:** Dynamic NestJS module with bucket and DI token options
**When to use:** Every service needing S3 storage

```typescript
// packages/foundation/src/storage/storage.module.ts
@Module({})
export class StorageModule {
  static forRootAsync(options: StorageModuleOptions): DynamicModule {
    return {
      module: StorageModule,
      imports: [TerminusModule],
      providers: [...storageProviders(options)],
      exports: [TerminusModule, options.token, STORAGE_HEALTH],
    };
  }
}
```

### Pattern 2: StoragePort interface

**What:** Abstraction over S3 operations with bucket baked in
**When to use:** All storage operations

```typescript
// packages/foundation/src/storage/storage.interfaces.ts
export interface StoragePort {
  upload(key: string, body: Buffer | Readable, contentType: string): Promise<void>;
  download(key: string): Promise<Buffer>;
  delete(key: string): Promise<void>;
  exists(key: string): Promise<boolean>;
  getSignedUrl(key: string, expiresInMs: number): Promise<string>;
}

export interface StorageModuleOptions {
  bucket: string;
  token: symbol;
}
```

### Pattern 3: Per-service wrapper module (mirrors CacheModule usage in sender)

**What:** Service-level module in infrastructure/storage/ that configures StorageModule
**When to use:** Each app that needs storage

```typescript
// apps/parser/src/infrastructure/storage/parser-storage.module.ts
import { Module, type DynamicModule } from '@nestjs/common';
import { StorageModule } from '@email-platform/foundation';
import { PARSER_STORAGE } from '../../parser.constants';

@Module({})
export class ParserStorageModule {
  static forRootAsync(): DynamicModule {
    return StorageModule.forRootAsync({ bucket: 'parser', token: PARSER_STORAGE });
  }
}
```

### Pattern 5: S3 Client construction with endpoint URL

**What:** Build S3Client from STORAGE_* env vars
**When to use:** In storageProviders factory

```typescript
// In storage.providers.ts
const s3ClientProvider: Provider = {
  provide: S3_CLIENT,
  inject: [ConfigService],
  useFactory: (config: ConfigService): S3Client =>
    new S3Client({
      endpoint: `http://${config.get<string>('STORAGE_ENDPOINT')}:${config.get<number>('STORAGE_PORT')}`,
      region: config.get<string>('STORAGE_REGION'),
      credentials: {
        accessKeyId: config.get<string>('STORAGE_ACCESS_KEY')!,
        secretAccessKey: config.get<string>('STORAGE_SECRET_KEY')!,
      },
      forcePathStyle: true,
      requestChecksumCalculation: 'WHEN_REQUIRED',
      responseChecksumValidation: 'WHEN_REQUIRED',
    }),
};
```

### Anti-Patterns to Avoid
- **Exposing raw S3Client:** D-04 forbids this. Only StoragePort abstraction.
- **Passing bucket per call:** D-01 requires bucket baked in at config time.
- **Environment branching (MinIO vs Garage):** D-15 forbids this. Same config, same code.
- **Using `z.coerce.boolean()` or `.default()` in schemas:** env-schema skill forbids this.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| Presigned URLs | Manual HMAC signing | @aws-sdk/s3-request-presigner | Security-critical, complex signing logic |
| S3 protocol | HTTP calls to S3 API | @aws-sdk/client-s3 | Handles retries, checksums, streaming, error parsing |
| Health check | Custom HTTP HEAD call | S3Client.send(HeadBucketCommand) | Already handles auth, endpoint resolution |
| Content streaming | Manual chunked reads | GetObjectCommand response.Body | SDK handles backpressure, errors |

## Common Pitfalls

### Pitfall 1: Checksum incompatibility with Garage/MinIO
**What goes wrong:** AWS SDK v3 >= 3.729.0 sends CRC32 checksums by default. Garage/MinIO return 501 Not Implemented.
**Why it happens:** AWS added mandatory checksums for data integrity; S3-compatible stores don't all support them.
**How to avoid:** Set `requestChecksumCalculation: 'WHEN_REQUIRED'` AND `responseChecksumValidation: 'WHEN_REQUIRED'` on S3Client.
**Warning signs:** 501 errors on PutObject/GetObject. [CITED: https://community.cloudflare.com/t/aws-sdk-client-s3-v3-729-0-breaks-uploadpart-and-putobject-r2-s3-api-compatibility/758637]

### Pitfall 2: Virtual-host style URLs with local storage
**What goes wrong:** SDK tries `bucket.localhost:9000` which doesn't resolve.
**Why it happens:** AWS SDK defaults to virtual-hosted style for S3.
**How to avoid:** Set `forcePathStyle: true` -- uses `localhost:9000/bucket` instead.
**Warning signs:** DNS resolution failures, ENOTFOUND errors. [CITED: https://garagehq.deuxfleurs.fr/documentation/reference-manual/s3-compatibility/]

### Pitfall 3: S3Client not destroyed on shutdown
**What goes wrong:** Open HTTP connections prevent clean process exit.
**Why it happens:** S3Client maintains an HTTP connection pool internally.
**How to avoid:** `S3ShutdownService` implements `OnApplicationShutdown` and calls `client.destroy()`.
**Warning signs:** Process hangs on SIGTERM, connection leaks in tests. [ASSUMED]

### Pitfall 4: Multiple S3Client instances per service
**What goes wrong:** Each `forRootAsync()` call creates a new S3Client and health indicator.
**Why it happens:** StorageModule is not global; multiple imports (service bucket + reports bucket) each construct providers.
**How to avoid:** Extract S3Client into a separate internal provider that is shared. Or accept one client per module (simpler, minimal overhead since each service has at most 2 buckets).
**Warning signs:** Duplicate health indicators, excessive connections.

### Pitfall 5: Endpoint URL construction
**What goes wrong:** Missing `http://` prefix or incorrect port concatenation.
**Why it happens:** StorageSchema has separate STORAGE_ENDPOINT and STORAGE_PORT fields.
**How to avoid:** Construct URL in provider: `` `http://${endpoint}:${port}` ``. Consider whether HTTPS is needed in production (Garage behind Traefik uses HTTP internally).
**Warning signs:** Connection refused, TLS errors. [ASSUMED]

### Pitfall 6: download() returning Readable vs Buffer
**What goes wrong:** GetObjectCommand response.Body is a Readable stream, not a Buffer.
**Why it happens:** AWS SDK v3 returns streaming responses by default in Node.js.
**How to avoid:** Convert to Buffer: `Buffer.from(await response.Body.transformToByteArray())` or use `transformToByteArray()`.
**Warning signs:** Type errors, empty downloads. [ASSUMED]

## Code Examples

### S3Client creation
```typescript
// Source: AWS SDK v3 docs + Garage compatibility requirements
import { S3Client } from '@aws-sdk/client-s3';

const client = new S3Client({
  endpoint: 'http://localhost:9000',
  region: 'us-east-1',
  credentials: {
    accessKeyId: 'minioadmin',
    secretAccessKey: 'minioadmin',
  },
  forcePathStyle: true,
  requestChecksumCalculation: 'WHEN_REQUIRED',
  responseChecksumValidation: 'WHEN_REQUIRED',
});
```

### Upload file
```typescript
// Source: AWS SDK v3 docs
import { PutObjectCommand } from '@aws-sdk/client-s3';

await client.send(new PutObjectCommand({
  Bucket: this.bucket,
  Key: key,
  Body: body,
  ContentType: contentType,
}));
```

### Download file
```typescript
// Source: AWS SDK v3 docs
import { GetObjectCommand } from '@aws-sdk/client-s3';

const response = await client.send(new GetObjectCommand({
  Bucket: this.bucket,
  Key: key,
}));
const bytes = await response.Body!.transformToByteArray();
return Buffer.from(bytes);
```

### Check if object exists
```typescript
// Source: AWS SDK v3 docs
import { HeadObjectCommand } from '@aws-sdk/client-s3';

try {
  await client.send(new HeadObjectCommand({ Bucket: this.bucket, Key: key }));
  return true;
} catch (error) {
  if ((error as { name: string }).name === 'NotFound') return false;
  throw error;
}
```

### Generate presigned URL
```typescript
// Source: AWS SDK v3 docs
import { getSignedUrl } from '@aws-sdk/s3-request-presigner';
import { GetObjectCommand } from '@aws-sdk/client-s3';

const url = await getSignedUrl(
  client,
  new GetObjectCommand({ Bucket: this.bucket, Key: key }),
  { expiresIn: Math.ceil(expiresInMs / 1000) },
);
```

### Health check via HeadBucket
```typescript
// Source: AWS SDK v3 docs
import { HeadBucketCommand } from '@aws-sdk/client-s3';

async isHealthy(key: string): Promise<HealthIndicatorResult> {
  const indicator = this.healthIndicatorService.check(key);
  try {
    await this.client.send(new HeadBucketCommand({ Bucket: this.bucket }));
    return indicator.up();
  } catch {
    return indicator.down({ message: S3_HEALTH_CHECK.DOWN_MESSAGE });
  }
}
```

### Shutdown service
```typescript
import { S3Client } from '@aws-sdk/client-s3';

@Injectable()
export class S3ShutdownService implements OnApplicationShutdown {
  constructor(@Inject(S3_CLIENT) private readonly client: S3Client) {}

  async onApplicationShutdown(): Promise<void> {
    this.client.destroy();
  }
}
```

## Open Questions

1. **S3Client sharing across multiple StorageModule instances**
   - What we know: A service importing both ParserStorageModule and ReportsStorageModule creates 2 S3Client instances
   - What's unclear: Whether to share one client or accept duplication (at most 2 per service)
   - Recommendation: Accept duplication -- simpler, matches CacheModule pattern (one Redis per CacheModule), at most 2 clients per service is negligible overhead

2. **STORAGE_BUCKET env var vs per-service bucket names**
   - What we know: STORAGE_BUCKET='email-platform' exists in .env. Per-service buckets are 'parser', 'sender', 'reports'
   - What's unclear: Whether STORAGE_BUCKET is the "default" bucket or a prefix, or unused
   - Recommendation: Ignore STORAGE_BUCKET for now -- per-service bucket names are hardcoded in service modules per D-07. STORAGE_BUCKET may be used for legacy or default cases

3. **Health indicator: which bucket to check?**
   - What we know: A service may have 2 buckets (own + reports). Health check uses HeadBucket.
   - What's unclear: Check one bucket or both?
   - Recommendation: Health indicator checks a configurable bucket (the main service bucket). Reports health is optional.
