## Summary

Plan 02 was originally scoped for Diun configuration, but the approach was pivoted mid-execution to CI push-based deploy via Coolify API. All infrastructure goals were achieved through an alternative path.

## What Was Done

- Added deploy job to `.github/workflows/docker-build.yml` — runs after all images are built

## Decisions

| Decision | Rationale |
|----------|-----------|
| CI push-based over Diun | Diun sends 1 webhook per image (6 deploys). CI calls once after all builds. Cleaner, no infrastructure overhead. |
| Composite action for deploy | Keeps workflow clean, secrets not visible in workflow file |
| One Garage per environment | Isolation between dev and prod data |
| Garage WebUI per environment | No CLI/SSH needed for bucket management |
| S3 client deferred | No code path uses S3 yet. Full storage module planned as next phase. |
