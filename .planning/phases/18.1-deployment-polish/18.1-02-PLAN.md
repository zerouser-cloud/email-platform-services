  <action>
Two independent cleanup tasks:

**A. Remove GitHub Manual Webhook:**

1. Go to https://github.com/zerouser-cloud/email-platform-services/settings/hooks
2. Find the webhook pointing to Coolify (URL contains `192.168.1.25:8000` or the Coolify manual webhook endpoint)
3. **Delete it** -- Diun replaces push-based deploys, this webhook causes double deploys
4. Confirm deletion

**B. Create Garage S3 Bucket and Keys:**

1. In Coolify, navigate to the Garage service
2. Access Garage admin UI or use Garage CLI to:
   - Create a bucket named `email-platform` (or `email-platform-dev` and `email-platform-prod` for separation)
   - Create an API key pair (access key + secret key)
3. In Coolify, update environment variables for BOTH dev and prod environments:
   - `STORAGE_ACCESS_KEY=<the-created-access-key>`
   - `STORAGE_SECRET_KEY=<the-created-secret-key>`
   - `STORAGE_BUCKET=email-platform` (or env-specific bucket name)
   - Verify `STORAGE_ENDPOINT` points to Garage's S3 endpoint
4. Redeploy affected services (parser, notifier -- the ones that use S3)

**Note:** If Garage is not yet needed for current functionality (no file upload features implemented), the S3 setup can be deferred. The TBD values won't cause crashes if no code path actually calls S3. User decides.
