# Phase 18.1: Deployment Polish

## Goal
Довести автодеплой до production-ready состояния. Исправить проблемы обнаруженные в фазе 18.

## Issues to Fix

### 1. Diun множественные webhook вызовы
- **Проблема:** Diun обнаруживает 6 образов → вызывает Coolify API 6 раз → конкурентные деплои → один падает
- **Решение:** Исследовать как Coolify обрабатывает множественные deploy запросы. Возможно использовать debounce или tag-based deploy.

### 2. Diun мониторит все контейнеры
- **Проблема:** WATCHBYDEFAULT=true мониторит 27 контейнеров включая postgres, redis, traefik, coolify сам
- **Решение:** Переключить на WATCHBYDEFAULT=false + добавить label `diun.enable=true` к нашим контейнерам в docker-compose.prod.yml

### 3. Prod auto-deploy не настроен
- **Проблема:** Diun webhook URL содержит только dev UUID
- **Решение:** Добавить prod UUID или настроить tag-based deploy

### 4. GitHub Manual Webhook cleanup
- **Проблема:** GitHub webhook всё ещё активен → двойные деплои
- **Решение:** Удалить webhook из GitHub repo settings

### 5. Убрать test version v2
- **Проблема:** Тестовый `version: 'v2'` в health endpoint — не нужен в проде
- **Решение:** Удалить или заменить на динамическую версию из package.json

### 6. Garage S3 (bucket + keys)
- **Проблема:** STORAGE_ACCESS_KEY и STORAGE_SECRET_KEY = TBD
- **Решение:** Создать bucket и ключи в Garage для dev и prod

## Dependencies
- Phase 18 completed
- Coolify API token working
- Diun running
